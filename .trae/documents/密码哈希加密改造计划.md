# 密码哈希加密改造计划

## 概述

将聊天室系统中所有密码存储从明文改为哈希加密，防止数据库泄露导致密码泄漏。

## 当前状态分析

* **数据库表**：`user_msg` 表中的 `password` 列以 `TEXT` 类型存储明文密码

* **注册流程**：客户端发来明文密码 → 服务器直接 `INSERT INTO user_msg (user_name, password) VALUES(?, ?)`

* **登录流程**：客户端发来明文密码 → 服务器 `SELECT ... WHERE user_name = ? AND password = ?` 直接比较

* **修改密码**：更新时同样以明文条件 `WHERE user_name = ? AND password = ?` 验证旧密码

* **影响的服务器文件**：

  * [server.py](file:///f:/Python%20Programs/Chat%20Room/server.py)（主服务器）

* **客户端文件**：客户端仍然发送明文密码（通过 SSL 加密传输），密码哈希在服务端完成，客户端无需修改

* **测试/工具文件**：[test\_signup.py](file:///f:/Python%20Programs/Chat%20Room/test_signup.py) 和 [check\_db.py](file:///f:/Python%20Programs/Chat%20Room/check_db.py) 需要同步更新

## 方案选择

使用 Python 内置库 `hashlib` 的 **PBKDF2-HMAC-SHA256** 进行密码哈希，原因：

1. 内置库，无需 pip 安装任何额外依赖
2. PBKDF2 是 NIST 推荐的标准密码派生函数
3. 可配置迭代次数（salt + 多次迭代），有效抵御暴力破解和彩虹表攻击

## 具体变更

### 1. server.py — 哈希工具函数

在文件头部 `import` 部分添加 `import hashlib, secrets`，并添加以下工具函数：

```python
import hashlib
import secrets

def hash_password(password: str) -> str:
    """使用 PBKDF2-SHA256 对密码进行哈希，返回 format: iterations:salt:hash"""
    salt = secrets.token_hex(32)  # 64字符十六进制salt
    iterations = 600000
    pwd_hash = hashlib.pbkdf2_hmac('sha256', password.encode(), salt.encode(), iterations)
    return f"{iterations}:{salt}:{pwd_hash.hex()}"

def verify_password(password: str, stored_hash: str) -> bool:
    """验证密码是否匹配存储的哈希"""
    try:
        iterations, salt, pwd_hash = stored_hash.split(':', 2)
        new_hash = hashlib.pbkdf2_hmac('sha256', password.encode(), salt.encode(), int(iterations))
        return new_hash.hex() == pwd_hash
    except (ValueError, AttributeError):
        return False
```

### 2. server.py — 注册时哈希密码

**位置**：`recv_msg` 方法中的 `login_msg[0] == "sign up"` 分支（当前第 475\~506 行）

**变更**：在 `INSERT` 之前对 `login_msg[2]`（密码）进行哈希

```python
# 改动前
sql = "INSERT INTO user_msg (user_name,password,icon_data) VALUES(?,?,?)"
cursor.execute(sql, [login_msg[1], login_msg[2], icon_bytes])

# 改动后
hashed_pwd = hash_password(login_msg[2])
sql = "INSERT INTO user_msg (user_name,password,icon_data) VALUES(?,?,?)"
cursor.execute(sql, [login_msg[1], hashed_pwd, icon_bytes])
```

### 3. server.py — 登录时验证哈希密码

**位置**：`recv_msg` 方法中的 `login_msg[0] == "login"` 分支（当前第 435\~474 行）

**变更**：先查询用户，获取存储的哈希，再用 `verify_password()` 比较

```python
# 改动前
sql = "SELECT user_name, password, icon_data, uid FROM user_msg WHERE user_name = ? and password = ?"
cursor.execute(sql, [login_msg[1], login_msg[2]])
result = cursor.fetchall()

# 改动后
sql = "SELECT user_name, password, icon_data, uid FROM user_msg WHERE user_name = ?"
cursor.execute(sql, (login_msg[1],))
result = cursor.fetchall()
if result and verify_password(login_msg[2], result[0][1]):
    # 登录成功
    ...
else:
    # 登录失败
```

### 4. server.py — 更新密码时验证旧密码

**位置**：`send_msg` 方法中的 `message[1][0] == "update"` 分支（当前第 224\~287 行）

**变更**：将旧密码的明文比较改为哈希验证

```python
# 改动前
sql = "SELECT user_name, password FROM user_msg WHERE user_name = ?"
cursor.execute(sql, (message[1][2][0],))
# 然后通过 result 是否存在来判断旧密码是否正确（实际逻辑是用旧信息查重名后做 UPDATE WHERE user_name=? and password=?)

# 改动后
# 获取当前存储的密码哈希
sql = "SELECT user_name, password FROM user_msg WHERE user_name = ?"
cursor.execute(sql, (message[1][1][0],))  # message[1][1][0] 是旧用户名
row = cursor.fetchone()
if row and verify_password(message[1][1][1], row[1]):  # message[1][1][1] 是旧密码
    # 旧密码验证通过，允许更新
    new_hashed = hash_password(message[1][2][1])  # 新密码哈希
    sql = "UPDATE user_msg SET user_name=?, password=?, icon_data=? WHERE user_name=?"
    cursor.execute(sql, [message[1][2][0], new_hashed, ..., message[1][1][0]])
```
